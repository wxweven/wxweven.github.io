Git配置多个SSH-Key

发表于 2019-10-25 | 分类于 Git
| 字数统计 231 字 | 阅读时长 1 分钟

为了安全性，个人的github和公司的gitlab需要配置不同的SSH-Key。具体步骤如下：

1. 切换到系统的SSH目录

2. 为个人的github生成SSH-Key（若还没有）

然后，前往github添加SSH公钥。

3. 为公司的gitlab生成SSH-Key（若还没有）

然后，前往gitlab添加SSH公钥。

4. 添加配置文件（若当前还没有config文件）

5. 为配置文件config添加如下内容

#测试

然后可以用同样方式测试公司的gitlab。
